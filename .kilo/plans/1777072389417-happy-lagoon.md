# Optimization Plan: Data Collector Enhancements

## Current Issues

1. **Ticker extraction is unreliable** - Current regex only matches parentheses `(SKRN)` but IDX uses bracket format `[SKRN ]` with trailing space
2. **Single PDF per announcement** - Only the first PDF link is captured; some announcements have multiple PDFs that are being skipped
3. **Selector strategy doesn't leverage known base element** - The user provided a specific base CSS selector that could provide more stable parsing

---

## Objective

Improve data collection reliability by:
- Extracting tickers from the `[TICKER ]` pattern (brackets + optional trailing space)
- Capturing ALL PDF links within each announcement container
- Using the confirmed base element structure for more robust parsing

## Optimization Details

### 1. Ticker Extraction Improvement

**Current implementation** (`data_collector.py:611-623`):
```python
def _extract_ticker(self, company_text: str) -> str:
    import re
    match = re.search(r"\(([A-Z]{3,4})\)", company_text)
    if match:
        return match.group(1)
    return company_text.strip()[:10]
```

**Problem**: Only matches parentheses, not brackets.

**New implementation**:
```python
def _extract_ticker(self, company_text: str) -> str:
    """Extract stock ticker from announcement title using bracket pattern."""
    import re
    
    # IDX format: "Title [TICKER " (brackets with optional trailing space)
    # Examples: "[SKRN ]", "[MINE ]", "[ADRO]"
    patterns = [
        r"\[([A-Z]{3,4})\s?\]",  # [SKRN ] or [SKRN]
        r"\(([A-Z]{3,4})\)",      # (SKRN) - fallback
    ]
    
    for pattern in patterns:
        match = re.search(pattern, company_text)
        if match:
            return match.group(1)
    
    # Fallback: try to find any 3-4 letter uppercase sequence
    match = re.search(r"\b([A-Z]{3,4})\b", company_text)
    if match:
        return match.group(1)
    
    return company_text.strip()[:10] if company_text else "UNKNOWN"
```

### 2. Multiple PDF Collection

**New data structure** - maintain backward compatibility:

```python
announcement = {
    "ticker": "SKRN",
    "title": "...",
    "date": "2026-04-25",
    "pdf_url": "https://...first.pdf",           # STRING (first PDF, for backward compatibility)
    "all_pdf_urls": ["url1.pdf", "url2.pdf"],   # LIST of all PDFs (new field)
    "content": "combined text from all PDFs",
    "pdf_saved": True,
    "pdf_paths": ["path1.txt", "path2.txt"],    # optional: individual files
}
```

**Rationale**: The `financial_expert.py` expects `announcement_data["pdf_url"]` (line 119, 134, 141, 149, 273). Changing it to a list would require modifying `financial_expert.py` and potentially breaking existing consumers. Instead:
- Keep `pdf_url` as the first PDF URL (backward compatible)
- Add `all_pdf_urls` field to store the complete list
- Frontend `source` field (from `financial_expert` output) will continue to point to first PDF

**Implementation approach**:

A. In `_collect_todays_announcements()`:
   - Within each matched row, find ALL `a[href$='.pdf']` links
   - Extract all PDF URLs into a list `all_pdf_urls`
   - Set `pdf_url` = first URL in list (for backward compatibility)
   - Store both fields in the announcement dict

B. In the main processing loop in `run()` (lines 154-223):
   - Iterate over `all_pdf_urls` list for each announcement
   - For each PDF: download, extract text, append to `ann["content"]`
   - Save one combined `.txt` file per announcement (or optional individual files via `pdf_paths`)
   - Track success: `pdf_saved=True` if at least one PDF downloaded
   - No change needed to `pdf_url` field (already set to first PDF)

C. Update `_fallback_collect_pdfs()` to return proper structure:
   - Set `pdf_url` to first PDF (or empty string if none)
   - Set `all_pdf_urls` to list of all found PDFs

### 3. Base Element Selector Strategy

**User-provided base selector**:
```
#app > div.sticky-footer-container-item.--pushed > main > div > div > div.tab-content.disclosure-tab > div:nth-child(2) > div > div:nth-child(4)
```

**New optional selector strategy**:
Add a new selector in `_collect_todays_announcements()` that:
1. Queries for announcement containers using the base selector pattern
2. For each container:
   - Extract ticker from `h6` element text
   - Find all PDF links (`a[href$='.pdf']`) within container
   - Extract title from the h6 or nearby heading
   - Check for date context within container
3. Filter by today's date (same as existing logic)
4. Return structured announcements

**Why add as separate strategy (not replace)**:
- Existing strategy works for current structure; user says base selector is more specific
- Keep both; try base selector first (more reliable), fall back to existing if 0 results
- Pre-emptive: if IDX changes layout, old strategy may still work

**Implementation sketch**:
```python
async def _collect_using_base_selector(self, page, today_iso: str, today_day: str) -> list[dict]:
    """Collect announcements using the known base container element."""
    announcements = []
    
    # Base container per announcement
    base_selector = "#app div.sticky-footer-container-item.--pushed main div.tab-content.disclosure-tab div:nth-child(2) div > div"
    containers = await page.query_selector_all(base_selector)
    
    for container in containers:
        # Get ticker from h6
        h6_elem = await container.query_selector("h6")
        if not h6_elem:
            continue
        h6_text = await h6_elem.inner_text()
        ticker = self._extract_ticker(h6_text)
        
        # Get all PDF links in container
        pdf_links = await container.query_selector_all("a[href$='.pdf']")
        pdf_urls = []
        for link in pdf_links:
            url = await link.get_attribute("href")
            if url:
                pdf_urls.append(url)
        
        if not pdf_urls:
            continue
        
        # Title from h6 text
        title = h6_text.strip()
        
        # Date check: look for today in container text
        container_text = await container.inner_text()
        if today_day not in container_text:
            continue
        
        announcements.append({
            "ticker": ticker,
            "title": title,
            "date": today_iso,
            "pdf_urls": pdf_urls,  # LIST
        })
    
    return announcements
```

## Risk Mitigation

- **Backward compatibility**: Keep announcement dict keys consistent; only change `pdf_url` (single) to `pdf_urls` (list)
- **Fallback behavior**: If no PDF links found in container, skip gracefully
- **Empty ticker handling**: Use `_extract_ticker()` fallbacks to ensure no blank tickers
- **Error isolation**: Each PDF download error is logged but doesn't abort entire announcement
